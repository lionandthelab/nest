# Nest (네스트) UX 측정 프레임워크 스코어카드

> 대상: 한국 홈스쿨 운영 관리 앱 (Flutter 모바일 + 웹)
> 역할별 사용자: 관리자(HOMESCHOOL_ADMIN), 부모(PARENT), 교사(TEACHER), 초청교사(GUEST_TEACHER)
> 기준 문서: PRD MVP (02), 사용자 스토리 (08), IA/권한 (03), 브랜드 비전 (01)

## 1. 로그인/회원가입 (FR-01 인증/계정)

### 1.1 정량 지표

| 지표 | 측정 방법 | 상용화 기준선 | 최고 수준 |
|---|---|---|---|
| 태스크 완료율 (이메일 로그인) | 로그인 시도 대비 성공 비율 | >= 95% | >= 99% |
| 태스크 완료율 (소셜 로그인) | 소셜 인증 시도 대비 성공 비율 | >= 92% | >= 98% |
| 소요 시간 (이메일 로그인) | 화면 진입 ~ 대시보드 도달 | <= 15초 | <= 8초 |
| 소요 시간 (회원가입) | 화면 진입 ~ 프로필 생성 완료 | <= 60초 | <= 35초 |
| 오류율 | 잘못된 입력으로 인한 재시도 비율 | <= 8% | <= 3% |
| 탭 수 (이메일 로그인) | 이메일 입력 ~ 대시보드 | <= 4탭 | <= 3탭 |
| 탭 수 (소셜 로그인) | 소셜 버튼 ~ 대시보드 | <= 3탭 | <= 2탭 |
| 비밀번호 복구 완료율 | 복구 요청 대비 로그인 재성공 | >= 80% | >= 92% |

### 1.2 정성 지표

| 구분 | 점검 항목 | 기준 |
|---|---|---|
| Nielsen H1 시스템 상태 가시성 | 로그인 진행 중 로딩 인디케이터 표시 여부 | 필수 |
| Nielsen H5 오류 예방 | 이메일 형식 실시간 검증, 비밀번호 강도 표시 | 필수 |
| Nielsen H9 오류 복구 지원 | 오류 메시지가 원인과 해결 방법을 명시하는지 | 필수 |
| 감성 디자인 - 따뜻함 | 브랜드 톤("따뜻한 둥지") 반영 환영 문구 존재 | 권장 |
| 감성 디자인 - 신뢰 | 개인정보 처리 안내 링크 접근 가능 | 필수 |
| 접근성 | WCAG 2.1 AA: 색상 대비 4.5:1 이상, 입력 필드 라벨 | 필수 |

### 1.3 상용화 기준선

- **스토어 출시 가능**: 태스크 완료율 95% 이상, 소요시간 기준 충족, Nielsen H1/H5/H9 충족
- **최고 수준**: 생체인증/자동로그인 지원, 소셜 로그인 원탭 전환, 0 오류 상태 UX

### 1.4 우선순위: **P0** (진입 장벽 -- 실패 시 전체 앱 사용 불가)

- **최고 수준**: 딥링크 기반 원클릭 가입, 진행 단계 시각화, 역할별 맞춤 온보딩

### 2.4 우선순위: **P0** (생성/가입 실패 시 앱 핵심 기능 전체 차단)

## 3. 관리자 대시보드 (KPI 요약 카드)

### 3.1 정량 지표

| 지표 | 측정 방법 | 상용화 기준선 | 최고 수준 |
|---|---|---|---|
| 첫 의미 있는 화면 렌더링 (FMP) | 대시보드 카드 전체 로딩 완료 시간 | <= 2초 | <= 1초 |
| 정보 탐색 소요시간 | 특정 KPI 수치 확인까지 시간 | <= 5초 | <= 2초 |
| 오류율 | 데이터 로딩 실패/빈 카드 노출 비율 | <= 3% | <= 0.5% |
| 탭 수 (상세 탐색) | 요약 카드 ~ 상세 화면 도달 | <= 2탭 | <= 1탭 |
| 새로고침 빈도 | 사용자 수동 새로고침 횟수/세션 | <= 1회 | 0회 (실시간 반영) |

### 3.2 정성 지표

| 구분 | 점검 항목 | 기준 |
|---|---|---|
| Nielsen H1 시스템 상태 가시성 | 데이터 최종 갱신 시각 표시 | 필수 |
| Nielsen H8 미적 간결성 | 카드당 정보 과부하 없이 핵심 수치만 표시 (카드당 지표 1~2개) | 필수 |
| Nielsen H6 재인지 우선 | 카드 아이콘/색상으로 영역 즉시 식별 | 권장 |
| 감성 디자인 - 신뢰 | 수치 변동 시 방향 화살표/색상 변화로 추이 표현 | 권장 |
| 감성 디자인 - 명확성 | 빈 데이터 상태(학기 미생성 등)에서 다음 행동 안내 | 필수 |
| 접근성 | 카드 내 수치 스크린리더 읽기 순서 보장 | 필수 |

### 3.3 상용화 기준선

- **스토어 출시 가능**: FMP 2초 이내, 빈 상태 안내 존재, 핵심 KPI 4종 이상 표시
- **최고 수준**: 실시간 갱신, 카드 재배치/개인화, 트렌드 미니 그래프

### 3.4 우선순위: **P1** (운영 시작 후 일상 접점이나, 셋업 플로우 우선)

| Nielsen H3 사용자 통제 | 잘못된 배정 즉시 취소/수정 가능, 아카이브 학기 수정 차단 안내 | 필수 |

## 5. 시간표 스튜디오 (FR-07, 핵심 기능)

### 5.1 정량 지표

| 지표 | 측정 방법 | 상용화 기준선 | 최고 수준 |
|---|---|---|---|
| 태스크 완료율 (채팅 생성안 적용) | 프롬프트 입력 ~ 1건 이상 세션 적용 | >= 85% | >= 95% |
| 태스크 완료율 (수동 D&D 편성) | 세션 카드 이동 ~ 충돌 없이 배치 확정 | >= 90% | >= 98% |
| 태스크 완료율 (시간표 최종 확정) | 편성 시작 ~ 확정 발행 | >= 80% | >= 95% |
| 소요 시간 (채팅 생성안 수신) | 프롬프트 전송 ~ 결과 화면 표시 | <= 10초 | <= 4초 |
| 소요 시간 (첫 시간표 확정) | 학기 셋업 완료 ~ 첫 시간표 확정 (KPI 목표) | <= 30분 | <= 15분 |
| 충돌 검증 응답 시간 | 카드 이동 ~ 충돌 결과 피드백 | <= 300ms (NFR-02) | <= 150ms |
| 오류율 | 충돌 상태에서 확정 시도 비율 | <= 5% | <= 1% |
| 탭/조작 수 (세션 1건 배치) | 카드 선택 ~ 슬롯 배치 확인 | <= 3조작 (선택+드래그+확인) | <= 2조작 |
| 채팅 생성안 적용률 | 생성된 안 중 사용자가 적용한 비율 | >= 60% | >= 80% |
| 충돌 발생 건수 (확정 전) | 확정 직전 잔여 충돌 수 | 0건 (차단 필수) | 0건 |

### 5.2 정성 지표

| 구분 | 점검 항목 | 기준 |
|---|---|---|
| Nielsen H1 시스템 상태 | 채팅 생성 중 진행 상태, 충돌 표시 즉시 시각 피드백 (색상+아이콘) | 필수 |
| Nielsen H2 실세계 부합 | 시간표 그리드가 실제 주간 시간표 형태와 일치 | 필수 |
| Nielsen H3 사용자 통제 | 실행취소(Undo)/다시실행(Redo) 지원, 부분 적용/전체 적용 선택 | 필수 |
| Nielsen H5 오류 예방 | 동일 교사-동일 시간 중복 배치 물리적 차단, 반 중복 차단 | 필수 |
| Nielsen H7 유연성 | 채팅 모드/수동 모드 자유 전환 | 필수 |
| Nielsen H9 오류 복구 | 충돌 메시지에 "어떤 교사가 어떤 시간에 겹치는지" 구체 명시 | 필수 |
| Nielsen H10 도움말 | 첫 사용 시 튜토리얼/가이드 오버레이 | 권장 |
| 감성 디자인 - 명확성 | 채팅 생성안의 근거(왜 이렇게 배치했는지) 설명 표시 | 필수 |
| 감성 디자인 - 신뢰 | 확정 전 미리보기/최종 확인 단계 존재 | 필수 |
| 접근성 | 드래그앤드롭 대안 입력(키보드/탭 기반 이동), 색약 대응 충돌 표시 | 필수 |

### 5.3 상용화 기준선

- **스토어 출시 가능**: 채팅+D&D 양 모드 작동, 충돌 0건 확정 보장, 첫 시간표 30분 이내
- **최고 수준**: Undo/Redo 무제한, 생성안 근거 자연어 설명, 이전 학기 시간표 복제, 교사 가용시간 자동 반영

### 5.4 우선순위: **P0 최우선** (앱의 핵심 차별 가치 -- "학기 운영 준비 시간 40% 절감" 달성의 핵심)

### Missing Questions

1. **사용성 테스트 대상자 프로필 미정의** -- 홈스쿨 관리자/부모/교사 각각 몇 명을 대상으로 테스트할 것인지에 따라 통계적 유의성이 달라진다.
2. **측정 기간(Baseline 수립 기간) 미정의** -- 상용화 기준선 달성 여부를 판단하려면 최소 몇 주간의 데이터 수집이 필요한지 결정해야 한다.
3. **오프라인/저대역 환경 빈도** -- 교사가 수업 현장(교회, 문화센터 등)에서 WiFi 없이 사용하는 비율에 따라 오프라인 모드 우선순위가 변동된다.
4. **교사 가용시간 위반 정책 미확정** -- PRD 오픈이슈에 "차단"/"경고" 미결정으로 남아 있어, 시간표 스튜디오의 오류율/완료율 측정 기준이 달라진다.
5. **부모 업로드 권한 여부 미확정** -- PRD 오픈이슈로 남아있으며, 갤러리 업로드 지표의 대상 사용자 범위가 달라진다.

### Undefined Guardrails

1. **미디어 파일 크기 상한** -- 업로드 성공률 측정 시 "어디까지가 정상 범위인지" 기준 필요. 제안: 사진 20MB, 영상 500MB 상한.
2. **동시 접속 관리자 수** -- 시간표 스튜디오 동시 편집 시 충돌 우려. 제안: MVP는 단일 편집자 잠금(lock) 명시.
3. **SUS/NPS 최소 응답 수** -- 통계 유의성 확보를 위한 최소 표본 수 정의 필요. 제안: SUS 최소 12명, NPS 최소 30명.

### Scope Risks

1. **시간표 스튜디오 Undo/Redo** -- "최고 수준" 지표에 포함했으나, 구현 복잡도가 높아 MVP 범위 침범 위험. 명확히 "상용화 기준" vs "로드맵" 구분 필요.
2. **접근성 WCAG AA 전면 적용** -- 전 화면 적용은 대규모 작업. MVP에서는 핵심 플로우(로그인, 시간표 조회, 업로드) 우선 적용으로 범위 한정 권장.
