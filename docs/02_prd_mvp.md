# Nest PRD (MVP)

## 1) 문서 목적

Nest 홈스쿨링 플랫폼의 **행정 운영 프레임**과 **현장 기록 체계**를 MVP로 구현하기 위한 제품 요구사항을 정의한다.

## 2) 배경 및 문제

기존 홈스쿨 운영은 다음 문제가 반복된다.
- 학기/반/수업/교사 배정이 스프레드시트와 메신저에 분산됨
- 시간표 충돌(교사 중복, 반 중복, 시간 겹침) 발생
- 부모/교사가 같은 정보를 실시간으로 확인하기 어려움
- 수업 사진/영상이 개인 폰·개인 클라우드에 분산되어 관리됨

Nest는 이를 단일 플랫폼으로 통합한다.

## 3) 목표

### 비즈니스 목표
- 홈스쿨 운영의 디지털 표준화
- 학기 운영 준비 시간 40% 절감
- 부모 문의성 메시지 30% 감소(정보 접근성 향상)
- 수업 미디어 자산 90% 이상을 기관 Drive로 중앙화

### 사용자 목표
- 관리자: 채팅 프롬프트/드래그앤드롭으로 빠르게 시간표 확정
- 부모: 내 아이 반 시간표와 수업 갤러리를 즉시 확인
- 교사: 담당 수업 계획/활동 기록/미디어 업로드를 한 곳에서 처리

## 4) 사용자 및 역할

- 홈스쿨 관리자 (`HOMESCHOOL_ADMIN`)
- 부모 (`PARENT`)
- 교사 (`TEACHER`)
- 초청 교사 (`GUEST_TEACHER`)
- 운영 스태프 (`STAFF`, 옵션)

참고: 교사는 부모 계정이 겸직할 수 있다.

## 5) 핵심 도메인 구조

- 한 사용자는 로그인 후 하나 이상의 홈스쿨에 소속 가능
- 각 홈스쿨에는 여러 가정(Family)이 참여
- 각 가정은 보호자(아빠/엄마 등)와 아이(1명 이상)로 구성
- 학기(Term)마다 반(Class Group)을 운영
- 아이는 학기 단위로 반에 배정
- 반 시간표에는 여러 수업 세션(Session)이 배치
- 세션마다 메인 교사 1명 + 보조 교사 0명 이상 배정
- 미디어는 홈스쿨 관리자 연결 Google Drive에 업로드되어 권한 기반으로 조회

## 6) 기능 요구사항

### FR-01 인증/계정
- 이메일/소셜 기반 로그인
- 홈스쿨 초대 링크를 통한 참여
- 사용자 프로필(이름, 연락처, 역할)

### FR-02 홈스쿨 생성/가입
- 사용자는 홈스쿨 개설 가능
- 홈스쿨별 고유 코드/초대 링크 발급
- 홈스쿨 관리자 권한 위임 가능

### FR-03 가정 및 구성원 관리
- 가정 생성/수정/비활성화
- 구성원 유형: 아빠, 엄마, 보호자, 아이
- 아이 기본 정보(이름, 생년월일, 특이사항)

### FR-04 교사 등록 관리
- 부모를 교사로 지정 가능
- 외부 초청 교사 등록 가능
- 교사 프로필: 전문 분야, 가능 요일/시간, 연락처

### FR-05 학기/반 관리
- 학기 생성(시작일, 종료일, 상태)
- 학기별 반 생성(예: 새싹반, 열매반)
- 반 정원/담임/보조담임 지정
- 아이 반 배정 및 이동

### FR-06 수업 정의
- 수업 카탈로그 관리(국어, 수학, 자연탐구 등)
- 수업별 목표, 권장 연령, 기본 소요시간

### FR-07 시간표 스튜디오 (핵심)
- 관리자 UI에서 주간 시간표 편성
- **모드 A: 채팅 프롬프트 편성**
- 자연어 프롬프트로 편성안 생성
- 생성안의 충돌/경고/근거 설명 제공
- 반별/교사별 부분 적용 가능
- **모드 B: 수동 드래그앤드롭 편성**
- 세션 카드 드래그 이동
- 슬롯 변경 시 즉시 충돌 검증
- 교사 배정 카드 수동 변경
- 동일 시간 교사 중복/반 중복 차단

### FR-08 학부모 조회
- 내 아이가 속한 반의 시간표 조회
- 학기별 캘린더/주간 보기
- 수업 변경 알림

### FR-09 교사 운영
- 담당 수업의 학기 계획표 등록
- 차시별 학습 목표/준비물/활동 기록 입력
- 아이별 관찰 메모 열람/기록

### FR-10 활동 기록
- 학생 활동 로그(출석, 과제, 관찰)
- 반/수업/학생 기준 필터링

### FR-11 공지/커뮤니케이션
- 홈스쿨 공지 게시
- 반 단위 공지 게시
- 변경 이력 조회

### FR-12 Google Drive 연동 (관리자)
- 홈스쿨 관리자 Google Drive OAuth 연결
- 업로드 루트 폴더 지정
- 반별/학기별 하위 폴더 정책 자동 생성

### FR-13 교사용 사진/영상 업로드
- 모바일에서 사진 촬영/영상 촬영 후 바로 업로드
- 메타데이터 입력(아이, 반, 수업, 태그)
- 대용량 업로드 재시도/재개

### FR-14 갤러리 (부모/교사 공통)
- Google Drive 연동 갤러리 목록/상세 조회
- 필터: 학기/반/아이/수업/기간
- 권한 범위 내 조회만 허용

## 7) 비기능 요구사항

### NFR-01 보안/개인정보
- 개인정보 최소 수집
- 전 구간 TLS
- 민감 데이터 암호화 저장
- 접근 로그/Audit 로그 보관

### NFR-02 성능
- 주요 목록/시간표 조회 API p95 < 600ms
- 시간표 충돌 검증 응답 p95 < 300ms
- 갤러리 목록 조회 API p95 < 900ms

### NFR-03 가용성
- 월간 서비스 가용성 목표 99.5% (MVP)

### NFR-04 확장성
- 멀티 홈스쿨 테넌시 고려
- 학기/반/학생/미디어 증가 시 수평 확장 가능 구조

### NFR-05 미디어 신뢰성
- 업로드 실패 재시도
- 업로드 완료 후 Drive 파일 ID 보장 저장
- 업로드 처리 상태 추적(`PENDING`, `UPLOADING`, `COMPLETED`, `FAILED`)

### NFR-06 감사 가능성
- 학기 시간표 변경 이력 저장
- 교사 배정 변경 이력 저장
- 미디어 업로드/삭제/권한 변경 이력 저장

## 8) 정책 규칙

- 한 세션에는 메인 교사 1명 필수
- 보조 교사는 0명 이상 허용
- 메인/보조 교사는 동일인 중복 지정 불가
- 학기 외 기간에는 시간표 발행 불가
- 아카이브된 학기는 수정 불가(읽기 전용)
- 부모는 본인 자녀 권한 범위의 갤러리만 조회 가능

## 9) MVP 범위 / 제외 범위

### MVP 포함
- 홈스쿨/가정/아이/교사/학기/반/수업/시간표/활동로그
- 시간표 스튜디오(채팅 프롬프트 생성 + 수동 드래그앤드롭)
- Google Drive 연동 업로드 + 권한 기반 갤러리 조회
- 권한 기반 접근제어

### MVP 제외
- 결제/정산
- 화상수업
- 고급 AI 추천 엔진(개별 학습 추천 자동화)
- 다국어/다지역 운영

## 10) KPI (초기)

- 학기 개설 후 첫 시간표 확정까지 평균 리드타임
- 시간표 충돌 발생 건수(확정 전)
- 채팅 생성안 적용률
- 학부모 주간 시간표 조회율
- 교사 미디어 업로드 성공률
- 학부모 갤러리 주간 활성 비율

## 11) 오픈 이슈

- 부모도 업로드 권한을 가질지 여부(MVP는 조회 전용)
- 교사 가용시간 위반 정책을 "차단"/"경고" 중 최종 확정 필요
- Drive 저장 구조를 "학기/반/날짜" vs "반/아이/날짜" 중 선택 필요
