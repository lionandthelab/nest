# 시간표 스튜디오 설계 (채팅 + 드래그앤드롭)

## 1) 목표

학기별 반 시간표를 생성할 때, 관리자가 **채팅 프롬프트로 빠르게 초안 생성**하고 **수동 드래그앤드롭으로 즉시 보정**한 뒤, 충돌 없이 확정할 수 있도록 한다.

## 2) 편성 UI 모드

### 모드 A: Chat Prompt Builder
- 입력: 자연어 프롬프트
- 예: "화/목 오전은 국어, 수학 중심으로, 새싹반은 점심 전 집중 수업으로 편성해줘"
- 출력
- 제안 시간표(반/슬롯/수업/교사)
- Hard 충돌 목록
- Soft 경고 목록
- 편성 근거(교사 가용시간, 과목 분산, 연속 수업 부담)
- 액션
- 전체 적용
- 반별 부분 적용
- 특정 세션만 적용

### 모드 B: Manual Drag & Drop Board
- 주간 그리드에 세션 카드 배치
- 카드 이동 시 실시간 충돌 체크
- 교사 카드 클릭으로 메인/보조 교사 교체
- 멀티 셀렉트로 일괄 이동

## 3) 충돌 규칙

### Hard Rule (저장 차단)
- 동일 `time_slot` + 동일 `class_group` 중복 세션 금지
- 동일 `time_slot` + 동일 `teacher` 중복 배정 금지
- 한 세션의 메인 교사는 정확히 1명
- 메인/보조 동일인 중복 금지

### Soft Rule (경고)
- 교사 가용시간 밖 배정
- 동일 과목 과다 연속 배치
- 동일 교사 연속 과다 배정(피로도 경고)

## 4) 처리 흐름

### A) 채팅 프롬프트 생성 흐름
1. 프롬프트 입력
2. 제안안 생성 (`proposal`)
3. Hard/Soft 검증
4. 사용자 검토
5. 부분/전체 적용
6. 커밋 전 최종 검증

### B) 수동 편집 흐름
1. 카드 드래그 이동
2. 즉시 검증
3. 충돌 시 롤백 또는 대안 슬롯 제시
4. 저장(초안)
5. 학기 시간표 확정

## 5) API 초안

### `POST /api/v1/terms/{termId}/timetable/assistant/generate`
- 입력: 프롬프트, 제약조건(교사 우선순위, 고정 수업 등)
- 출력: `proposal_id`, 제안 세션, 충돌/경고, 설명

### `POST /api/v1/terms/{termId}/timetable/proposals/{proposalId}/apply`
- 입력: 적용 대상(전체/반별/세션별)
- 출력: 반영 결과, 미반영 항목

### `PATCH /api/v1/class-sessions/{sessionId}`
- 입력: `time_slot_id`, `teacher_assignments`
- 출력: 수정 세션, 검증 결과

### `POST /api/v1/terms/{termId}/timetable/validate`
- 입력: 변경 세션 배열
- 출력: `is_valid`, `hard_conflicts[]`, `soft_warnings[]`

### `POST /api/v1/terms/{termId}/timetable/commit`
- 입력: `timetable_version`
- 처리: 원자 트랜잭션 저장
- 출력: 확정 버전

## 6) 동시성 제어

- 시간표 편성은 `term_id` 단위 낙관적 잠금(version) 사용
- 커밋 시 버전 불일치면 `409 Conflict` 반환
- 다중 관리자 편집 시 최신 스냅샷 재로딩 유도

## 7) UX 요구사항

- 채팅 모드와 수동 모드를 같은 화면에서 탭 전환
- 충돌 슬롯 즉시 색상 표시
- 충돌 원인을 자연어로 안내
- 예: "김선생님은 화요일 10:00 슬롯에 이미 새싹반 수업이 배정되어 있습니다."
- "대체 시간 추천"과 "대체 교사 추천" 버튼 제공

## 8) 감사 및 이력

- 채팅 프롬프트 원문 및 생성 결과 버전 저장
- 편성 확정 이벤트 기록
- 교사 배정 변경 전/후 비교 저장
- 누가 언제 어떤 세션을 수정했는지 조회 가능

## 9) 테스트 시나리오 (최소)

- 채팅 생성안에 반 중복이 있을 경우 적용 차단
- 채팅 생성안 부분 적용 시 나머지 시간표 무결성 유지
- 드래그 이동 시 교사 중복 충돌 즉시 표시
- 메인 교사 누락 시 저장 차단
- 교사 가용시간 위반 시 경고 노출
- 동시 커밋 시 버전 충돌 처리
